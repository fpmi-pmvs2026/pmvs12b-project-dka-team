# Дополнительная спецификация

## Ограничения

- Приложение зависит от внешнего API (NewsAPI) и качества интернет-соединения.
- Для сборки требуется файл `code/keystore.properties` с `NEWS_API_KEY`.
- Поддерживаемая версия Android: от `minSdk 24`.
- При превышении лимитов NewsAPI часть запросов может быть отклонена.

## Требования к безопасности

- API-ключ не должен попадать в публичный репозиторий.
- В репозитории хранится только шаблон конфигурации, секреты передаются локально.
- Все сетевые обращения выполняются по HTTPS.
- Для Android 13+ уведомления включаются только после явного разрешения пользователя.

## Требования к надежности

- Локальное хранение подписок и статей в Room обеспечивает работу при временной недоступности сети.
- Фоновое обновление выполняется через WorkManager, устойчиво к перезапуску приложения.
- Повторное добавление существующей подписки/статьи не дублирует запись (on conflict ignore).
- При удалении подписки связанные статьи удаляются каскадно.

## Требования к производительности

- Загрузка изображений вынесена в Coil с кешированием.
- Отображение списков реализовано через `LazyColumn`/`LazyRow`.
- Обновление данных выполняется асинхронно, без блокировки UI-потока.

## Масштабируемость и сопровождение

- Слои `presentation/domain/data` позволяют изолировать изменения.
- Use case обеспечивают явное разделение бизнес-логики.
- DI через Hilt упрощает тестирование и замену реализаций.
