# Презентация проекта

## Ссылка на презентацию

- [Открыть презентацию (PDF)](presentation/presentation.pdf)

Рекомендуемый формат: Google Slides / PowerPoint Online с доступом по ссылке.

## Что должно быть в презентации

1. **Краткая цель проекта**
   - какую задачу решает приложение;
   - кто целевой пользователь.

2. **Распределение задач в команде**
   - участники и роли;
   - какие задачи выполнял каждый.

3. **Функциональные требования**
   - ключевые функции приложения;
   - use case и пользовательские сценарии.

4. **Архитектура и структура проекта**
   - слои `presentation/domain/data`;
   - основные модули и зависимости.

5. **Схема базы данных**
   - сущности, ключи, связи;
   - объяснение выбранной структуры хранения.

6. **Организация работы с репозиторием**
   - git-flow/ветвление;
   - pull request, code review, CI/CD (если использовались).

7. **Итоги и планы**
   - что реализовано;
   - что можно улучшить дальше.
