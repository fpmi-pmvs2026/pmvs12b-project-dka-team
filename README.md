# NewsAppDemo (pmvs12b-project-dka-team)

Учебный Android-проект для получения новостей через [NewsAPI](https://newsapi.org/).
Приложение позволяет подписываться на темы, просматривать статьи, вручную и автоматически обновлять данные, а также настраивать язык, интервал обновления и уведомления.

## Что умеет приложение

- Подписка на произвольные темы и выбор активных подписок.
- Загрузка новостей по выбранным подпискам.
- Просмотр карточек статей (изображение, источник, дата, описание).
- Переход к статье в браузере и шаринг ссылки.
- Очистка локально сохранённых статей.
- Экран настроек:
  - язык поиска новостей (`English`, `Русский`, `Français`, `Deutsch`);
  - интервал фонового обновления;
  - уведомления о новых статьях;
  - обновление только по Wi-Fi.

## Технологии и стек

- Kotlin, Jetpack Compose, Material 3
- Clean Architecture + MVVM
- Hilt (DI)
- Retrofit + Kotlinx Serialization (сеть)
- Room (локальная база данных)
- DataStore (настройки)
- WorkManager (фоновое обновление)
- Coil (загрузка изображений)

## Структура репозитория

В этом репозитории основной код приложения находится в git-сабмодуле:

- `code` — Android-проект (`:app` модуль)

Если вы клонируете репозиторий впервые, обязательно инициализируйте сабмодуль.

## Требования

- Android Studio (актуальная версия)
- JDK 11
- Android SDK:
  - `compileSdk = 36`
  - `minSdk = 24`
  - `targetSdk = 36`
- Ключ API для [NewsAPI](https://newsapi.org/)

## Быстрый старт

### 1) Клонирование репозитория

```bash
git clone --recurse-submodules <URL_ЭТОГО_РЕПОЗИТОРИЯ>
cd pmvs12b-project-dka-team
```

Если репозиторий уже клонирован без сабмодулей:

```bash
git submodule update --init --recursive
```

### 2) Добавление API-ключа

В папке `code` создайте файл `keystore.properties`:

```properties
NEWS_API_KEY="YOUR_API_KEY"
```

> Важно: без этого файла приложение не соберётся, так как ключ используется в `BuildConfig`.

### 3) Запуск из Android Studio

1. Откройте папку `code` как Android-проект.
2. Дождитесь синхронизации Gradle.
3. Запустите конфигурацию `app` на эмуляторе или устройстве.

## Запуск и проверка через Gradle

Из директории `code`:

```bash
./gradlew assembleDebug
./gradlew test
./gradlew connectedAndroidTest
```

## Архитектура (кратко)

- `presentation` — UI на Compose, `ViewModel`, навигация.
- `domain` — бизнес-логика, use case, сущности, интерфейсы репозиториев.
- `data` — реализация репозиториев, сеть, БД, мапперы, фоновый worker.
- `di` — модули внедрения зависимостей (Hilt).

## Полезно знать

- Фоновое обновление запускается при старте приложения через `AppStartupManager` и `WorkManager`.
- Уведомления о новых статьях зависят от настройки в приложении и разрешения на Android 13+.
- Существующий `.gitmodules` указывает на сабмодуль `code` с исходниками приложения.
